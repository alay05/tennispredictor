## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| calibrated model outputs→artifact bundle | Saved artifacts must preserve provenance and must not be silently overwritten |
| repo-local filesystem→artifact loader | Loading persisted joblib/XGBoost files is only safe after manifest/path validation |
| calibrated predictions→segment diagnostics | Segment bins must remain explicit and stable across runs to keep comparisons trustworthy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Elevation of Privilege | `registry.py` loaders | mitigate | Validate repo-local paths, required files, feature version, split-manifest ID, and dependency metadata before any joblib or XGBoost model deserialization. |
| T-03-04-02 | Repudiation | `write_model_artifact_bundle` | mitigate | Use immutable `models/runs/<run_id>/` directories and fail on overwrite so every saved run remains auditable and append-only. |
| T-03-04-03 | Tampering | `metrics.py`, `reports.py` | mitigate | Persist explicit segment-bin definitions and derived CSV/JSON outputs so later runs cannot silently reinterpret ranking or confidence segments. |
</threat_model>
