<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frozen split manifest→candidate trainer | The trainer must not smuggle validation/test memberships into fit-time optimization |
| raw fitted model→calibration layer | Calibration expects a model already fit on disjoint data |
| calibrated probabilities→metrics surface | Probability-quality metrics are only meaningful if they evaluate the correct calibrated outputs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | `xgboost_model.py` | mitigate | Reserve only the trailing 15% of train memberships for fit-time `eval_set` and forbid the external validation split from entering early stopping. |
| T-03-03-02 | Tampering | `calibration.py` | mitigate | Fit the calibrator on validation rows only, persist the chosen method, and assert through tests that test rows are evaluated but never fit. |
| T-03-03-03 | Repudiation | `metrics.py` | mitigate | Return explicit scalar metric names, calibration bins, and ECE values from calibrated probabilities so later plans do not reconstruct or rename metrics ad hoc. |
</threat_model>
