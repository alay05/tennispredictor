## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| in-memory feature build→DuckDB feature tables | Persistence must not drop lineage, counts, or side orientation on the way to storage |
| future match rows→historical feature snapshots | Any hidden dependency here invalidates the phase’s leakage claims |
| later phases→stored feature tables | Training/backtest/live code must consume stored snapshots rather than recomputing state elsewhere |
