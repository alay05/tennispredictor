## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| canonical match lineage→match-stat lookup | Two distinct Sackmann files can reuse the same row number, so source lineage must be unique enough to select the right prior stat row |
| pre-match snapshots→differential rows | A corrupted stat snapshot propagates directly into FEAT-08 differential features |
| in-memory feature build→persisted feature tables | Persistence must prove it is carrying through the corrected stat identity rather than masking a lookup bug |
