<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| canonical match stats→aggregate state | Missing or incomplete Sackmann stats can be misinterpreted as real zeros |
| pair history→H2H features | A symmetric pair store must not double-count or leak the current match |
| player-side snapshots→rate differentials | Differential math must preserve missingness instead of inventing values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering | `state.py` stat aggregation | mitigate | Use only prior canonical stat rows, keep missing values explicit, and gate low-exposure histories with `stats_low_sample` instead of silent zero-fill. |
| T-02-03-02 | Repudiation | `schemas.py`, `differential.py` | mitigate | Persist counts and availability flags alongside every stat/H2H feature so later consumers can explain why a value is null or present. |
| T-02-03-03 | Spoofing | `state.py` H2H state | mitigate | Key H2H state symmetrically by canonical player pair and snapshot it before current-match updates so one player cannot inherit future pair results. |
</threat_model>
