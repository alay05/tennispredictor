<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| canonical Phase 1 tables→feature runner | Only canonical, ATP-only, lineage-bearing rows should enter the feature pipeline |
| ranking history→point-in-time lookup | A wrong lookup strategy can leak future rankings into historical matches |
| player-side snapshots→differential rows | Reconstructing differentials from raw matches would bypass the single-source-of-truth contract |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Tampering | `ordering.py`, `runner.py` | mitigate | Centralize deterministic cohort ordering and assert same-round baseline behavior in `test_build_feature_snapshots_emits_pre_match_rank_and_context_before_cohort_updates`. |
| T-02-01-02 | Information Disclosure | `rankings.py` | accept | The phase uses public ranking data only; no secret-bearing inputs cross this boundary. |
| T-02-01-03 | Repudiation | `schemas.py`, `differential.py` | mitigate | Carry `feature_version`, match/player IDs, side orientation, and source lineage in every snapshot and row so downstream consumers never rebuild provenance ad hoc. |
</threat_model>
