## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| artifact bundle→replay harness | Loading saved model files is only safe after manifest, checksum, and path validation |
| persisted feature rows→replay outputs | Replay must preserve frozen row membership and row ordering instead of re-deriving time windows |
| replay outputs→later EV/report slices | Every row needs enough provenance to support later rejected/accepted decision records and backtest reports |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Elevation of Privilege | `src/tennisprediction/backtesting/replay.py` | mitigate | Load saved estimators only through `load_model_artifact_bundle`, so repo-local path, checksum, feature-version, and split-manifest validation happen before deserialization. |
| T-04-01-02 | Tampering | replay row selection | mitigate | Select replay rows strictly from the frozen split manifest and saved `feature_columns` order; do not re-derive test windows from dates inside the replay harness. |
| T-04-01-03 | Repudiation | replay outputs | mitigate | Emit artifact run ID, feature version, split manifest ID, and pinned source commit SHA on every replay row so later EV or reporting code cannot claim untraceable predictions. |
</threat_model>
