<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| upstream Git snapshot→local raw storage | External source content crosses into the repo-local raw data area here |
| raw files→manifest metadata | File bytes and provenance metadata must stay aligned for auditability |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Spoofing | `sackmann_fetcher.py` | mitigate | Require explicit commit SHA input and store the exact commit in the manifest; reject branch-name based acquisition. |
| T-01-02-02 | Tampering | raw snapshot files | mitigate | Compute and persist file SHA256 checksums in the manifest, then verify them whenever an existing snapshot is loaded. |
| T-01-02-03 | Information Disclosure | manifest metadata | mitigate | Record attribution/license/source metadata only; avoid embedding tokens, credentials, or non-repo secrets in manifest files. |
| T-01-02-04 | Repudiation | acquisition history | mitigate | Persist acquisition timestamp, upstream URL/path, and commit SHA so later canonical outputs can trace back to one raw snapshot. |
</threat_model>
