<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw CSV/parquet→validator | Untrusted upstream data crosses into typed validation here |
| validator→canonical candidate set | Only schema-approved, in-scope rows should pass this boundary |
| excluded rows→quarantine store | Audit data must retain provenance and exclusion reasons |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Tampering | `validation.py` | mitigate | Fail fast on missing/extra critical columns, type drift, and parse errors before returning any canonical candidates. |
| T-01-03-02 | Elevation of Privilege | `quarantine.py` | mitigate | Enforce ATP-only file and row rules centrally so excluded or non-ATP data cannot bypass quarantine through a loose caller. |
| T-01-03-03 | Repudiation | quarantine outputs | mitigate | Persist explicit reason codes and upstream lineage references for every quarantined row/file. |
| T-01-03-04 | Denial of Service | malformed raw inputs | mitigate | Validate incrementally from the manifest contract and surface bounded, actionable errors instead of cascading failures later in normalization. |
</threat_model>
